Feature Highlight: Time-based caching

One of the most commonly used Gramex features is time-based caching. You can cache any web page, API or query for a fixed duration using the cache expiry feature.

 

# To cache the request for 5 seconds

cache:

expiry:

duration: 5

 

 

This is useful in 3 situations:

  • A report is slow, but doesnâ€™t change frequently. Caching it for an hour speeds up the report for subsequent users.
  • A popular page is visited by millions of users. Caching it for one second slows down the rate of request to 1 per second, while caching the page for all other users.
  • An expensive query or a metered service on the cloud. Caching it forever ensures that all past queries are available, and the same request is not charged for again. 

You can learn more on the Gramex guide to caching.
